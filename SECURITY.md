# Security policy

## Supported versions

Security fixes are applied to the latest released version and the default
branch. Older versions may not receive patches.

## Report a vulnerability

Use GitHub's private vulnerability reporting or a private security advisory for
this repository. If that option is unavailable, contact the maintainer through
the repository owner's GitHub profile before publishing details.

Include the affected version, impact, minimal reproduction, and suggested
mitigation. Remove real prompts, credentials, authorization headers, private
paths, and repository content from the report.

Please do not open a public issue for an unpatched vulnerability. You can expect
an initial response within seven days.
