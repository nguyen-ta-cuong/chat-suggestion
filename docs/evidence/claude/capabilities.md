# Claude capability evidence

No runnable Claude Code executable was available in the observed development
environment. The adapter's current evidence is therefore limited to offline fake
executables and synthetic protocol profile descriptors. This document does not
claim a current Claude native editor API.

A future verified capability entry must record the exact public version/help
commands, a public API reference, a no-cost initialization handshake, and
fixture results for Unicode, resize, completion UI, hidden input, disposal, and
unknown output. Lifecycle hooks such as `UserPromptSubmit` may supply
post-submit context but cannot raise `inlineRender` without semantic pre-submit
editor ownership.

No credentials, private settings paths, prompts, sessions, or model calls are
part of the probe or evidence process.
