# Plan 0011 contract-change request

## Missing Pi distributable extension artifact

Plan 0011 verified Pi 0.80.6's documented local-package command against the
completed workspace. `pi install "$PWD/adapters/pi" --local --approve` records
the package, but a subsequent Pi startup fails before opening the editor because
the directory is not a loadable Pi extension module. The adapter package exports
library functions that require an injected `SuggestionBridge`; it does not ship
a user-loadable extension entry point that creates that bridge.

This plan cannot change runtime code or package metadata. The user guide
therefore documents the verified disposable fake-provider smoke extension at
`adapters/pi/smoke/offline-extension.ts`, but marks durable Pi installation as
unavailable for this release. A future integration change needs to provide a
loadable, documented Pi package entry that wires configuration, context,
provider, and `createPiSuggestionExtension`, then verify install, startup,
disable, and removal using an isolated Pi configuration.

Evidence (2026-07-11): Pi 0.80.6 accepted the local install command and later
reported `Failed to load extension ... Cannot find module .../adapters/pi`. The
test-created `.pi/settings.json` was removed immediately afterward.
